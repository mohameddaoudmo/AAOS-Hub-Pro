الأساسيات · Basics
ما هو ADB؟
Android Debug Bridge — الجسر بين جهازك والأندرويد
التعريف البسيط

ADB هو command-line tool بيخليك تتحكم في أي Android device أو Emulator من الـ PC. بتقدر تثبت APKs، تنسخ ملفات، تشغّل shell commands، تشوف الـ logs، وتعمل أي حاجة تقريباً في الجهاز.

ADB Architecture — 3 مكونات
ADB Client
PC / Terminal
⟷
ADB Server
Port 5037
⟷
ADB Daemon
adbd على الجهاز
📟 بتكتب الأوامر هنا 🔄 بيدير الاتصالات 📱 بيشتغل على الجهاز
ADB Client

البرنامج اللي بتشغله في الـ Terminal. لما بتكتب adb devices ده هو الـ client. بيتواصل مع الـ Server على port 5037.

ADB Server

بيشتغل في الـ background على جهازك. بيدير الاتصالات بين الـ Client والأجهزة المتصلة. بيبدأ تلقائياً لما تشغّل أي ADB command.

ADB Daemon (adbd)

بيشتغل على الـ Android device نفسه كـ background process. بيستقبل الأوامر من الـ Server وينفذها. ممكن يشتغل كـ root أو كـ normal user.

💡
فين ADB في الـ Android SDK؟

بتلاقيه في: Android/Sdk/platform-tools/adb على Windows أو ~/Library/Android/sdk/platform-tools/adb على Mac.

APK Commands
# ═══ تثبيت APK ═══
$ adb install app-debug.apk

# -r = reinstall مع الاحتفاظ بالبيانات
$ adb install -r app-debug.apk

# -d = downgrade (نسخة أقدم)
$ adb install -r -d app-old.apk

# -g = منح كل الـ permissions تلقائياً
$ adb install -r -g app.apk

# ═══ حذف تطبيق ═══
$ adb uninstall com.example.myapp

# حذف مع الاحتفاظ بالـ data و cache
$ adb uninstall -k com.example.myapp

# ═══ قائمة التطبيقات المثبتة ═══
$ adb shell pm list packages
$ adb shell pm list packages -3    # third-party فقط
$ adb shell pm list packages -s    # system فقط
$ adb shell pm list packages -d    # disabled فقط

# فلترة نتائج
$ adb shell pm list packages | grep google

# ═══ مسار الـ APK على الجهاز ═══
$ adb shell pm path com.example.myapp
package:/data/app/com.example.myapp-1/base.apk

# ═══ استخراج APK من الجهاز ═══
$ adb pull /data/app/com.example.myapp-1/base.apk ./extracted.apk

# ═══ تثبيت Multiple APKs (Split APKs) ═══
$ adb install-multiple base.apk split_config.arm64.apk

$ adb shell cmd car_service inject-vhal-event 0x11600204 0 45.0

$ adb shell cmd car_service inject-vhal-event 0x1140040a 0 0.8

⚠️ ADB Security Risks

ADB بيديك access كامل للجهاز. لو تركت USB Debugging مفتوح على جهاز حقيقي وربطته بشاحن عام، أي حاجة ممكن تحصل. دايماً:

  • اطفي USB Debugging لما مش بتشتغل
  • استخدم "Revoke USB debugging authorizations" بعد كل session
  • لا توصل بـ USB في أماكن مجهولة

Troubleshooting
حل المشاكل الشائعة
كل مشكلة ممكن تقابلها وحلها

❌ "device unauthorized"

الجهاز مش قبل الاتصال. الحل: افتح الجهاز وهتلاقي dialog "Allow USB Debugging?" → اضغط Allow. لو مش ظهرت: اسحب التوصيلة وأعد توصيلها.

$ adb kill-server
$ adb start-server
$ adb devices

⚠️ "no devices/emulators found"

مفيش جهاز متصل أو الـ server مش شاغل. تأكد إن USB Debugging مفعّل وإن الـ cable تمام.

$ adb kill-server && adb start-server
# على Linux: تأكد من udev rules
$ sudo udevadm control --reload-rules

🔌 Emulator لا يظهر في adb devices

شغّل الـ Emulator أولاً ثم انتظر دقيقة. لو مازال مش ظاهر:

$ adb connect localhost:5554
$ adb connect 127.0.0.1:5554
# Emulators بيستخدموا 5554, 5556, 5558...

🐌 ADB بطيء جداً

جرب TCP بدل USB، أو restart الـ server. على WiFi تأكد إن الجهاز والـ PC على نفس الـ network.

$ adb kill-server
$ adb start-server
# تحقق من السرعة
$ adb shell dd if=/dev/zero bs=1m count=100 | adb pull /dev/stdin /dev/null

❌ "INSTALL_FAILED_USER_RESTRICTED"

على بعض الأجهزة الصينية فيه إعداد زيادة. روح Settings → Security → اسمح بتثبيت تطبيقات من مصادر مجهولة، أو فعّل "USB Installation" في Developer Options.

⚠️ VHAL inject مش بيتنفذ

تأكد إنك على AAOS Emulator مش Emulator عادي. الـ car_service بيكون موجود بس على Android Automotive OS AVD.

# تحقق إن car_service موجود
$ adb shell dumpsys -l | grep car
car_service  ← لازم تظهر
🚫
لا تعمل adb root على الجهاز الحقيقي في Production

الـ adbd بيرفض الـ root على user builds (جهاز حقيقي). ده بس بيشتغل على userdebug وeng builds، يعني الـ Emulator والأجهزة التطويرية بس.
