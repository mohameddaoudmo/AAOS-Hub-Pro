01 — نظرة عامة

AAOS Architecture — الصورة الكاملة

02 — CarService

CarService — العقل المدبّر

الـ CarService هو System Service بيشتغل كـ process منفصل بامتيازات عالية. هو الوسيط الإلزامي بين كل App والـ VHAL. مش ممكن لأي app يكلم الـ VHAL مباشرةً — لازم يعدي من CarService.

📋 مسؤوليات CarService

  • تلقي طلبات الـ Apps عن طريق Binder IPC
  • التحقق من الـ Permissions قبل أي عملية
  • ترجمة الطلبات لـ VHAL property calls
  • إدارة الـ Property Change Listeners والـ Callbacks
  • Caching لقيم الـ Properties لتقليل calls للـ HAL
  • إدارة Vehicle Modes (Normal, Restricted, Parked)
  • Lifecycle management مع الـ car power state

🏗️ Sub-Services داخل CarService

  • CarPropertyService — يدير كل Vehicle Properties
  • CarHvacService — التحكم في التكييف
  • CarAudioService — إدارة الصوت والتوجيه
  • CarMediaService — إدارة Media Sessions
  • CarSensorService — السرعة والـ Gyroscope
  • CarPowerManagementService — إدارة الطاقة
  • CarPackageManagerService — فلترة الـ Apps
java — الاتصال بـ CarService من App // 1. الحصول على Car instance Car car = Car.createCar(context, new Car.CarServiceLifecycleListener() { @Override public void onLifecycleChanged(Car car, int event) { if (event == Car.CAR_WAIT_FOR_SERVICE_READY) { // CarService متاح — اطلب الـ Manager اللي تحتاجه onCarServiceReady(car); } } }); car.connect(); // 2. الحصول على Manager معين void onCarServiceReady(Car car) { CarPropertyManager propMgr = (CarPropertyManager) car.getCarManager(Car.PROPERTY_SERVICE); CarHvacManager hvacMgr = (CarHvacManager) car.getCarManager(Car.HVAC_SERVICE); // ... استخدم الـ managers }
⚠️ مهم جداً: Lifecycle Management

لازم دايماً تعمل car.disconnect() في الـ onDestroy() أو لما ما عدتش محتاج CarService. نسيان ده بيسبب memory leaks ومشاكل في الـ car process.

📐 Property Access Modes

Access Mode المعنى مثال Read/Write
READ قراءة فقط Vehicle Speed, Engine Temp Read Only

READ_WRITE قراءة وكتابة HVAC Temp, Seat Position Both

📊 Property Change Modes

Change Mode المعنى استخدام
STATIC مش بتتغير أبداً VIN, Make/Model, Fuel Type

CONTINUOUS Updates بـ rate ثابت (Hz) Speed (1-10Hz), GPS, Accel

04 — HAL Layer

HAL — Hardware Abstraction Layer

الـ HAL (Hardware Abstraction Layer) هو الطبقة اللي بتفصل الـ Android Framework عن الـ Hardware المحدد. الفكرة: الـ Framework بيتكلم مع interface موحّد، والـ OEM (صانع السيارة) بيعمل implementation خاص بيه.

🎯 ليه HAL موجود؟

  • فصل الـ Android code عن الـ Vendor code
  • كل OEM بيعمل implementation بتاعه بدون تعديل الـ AOSP code
  • يسمح بـ Android updates بدون تغيير الـ Vendor software
  • الـ HAL بيشتغل في process منفصل (isolated) لأمان أكتر

📡 أنواع HAL Interfaces

  • HIDL (HAL Interface Definition Language) — الجيل القديم قبل Android 11
  • AIDL (Android Interface Definition Language) — الجيل الجديد من Android 11+، نفس الـ Binder
  • الـ VHAL بالذات اتنقل من HIDL لـ AIDL في Android 13

java — كيف الـ Binder بيشتغل في AAOS // الـ App مش بيكلم CarService مباشرة // بيستخدم Proxy Object بيحول الـ method calls لـ Binder transactions // ما يحصل خلف الكواليس لما بتعمل: propMgr.getProperty(VehiclePropertyIds.PERF_VEHICLE_SPEED, 0); // 1. CarPropertyManager بيعمل: mService.getProperty(propertyId, areaId); // حيث mService هو ICarProperty (Binder proxy) // 2. الـ Binder Driver بيعمل: // - يحول الـ call لـ Parcel (serialized data) // - ينقله للـ CarService process // - ينفذ الـ call في thread من الـ Binder thread pool // 3. CarService بيرد: // - ينفذ getProperty() فعلياً // - يرجع الـ result كـ Parcel // - الـ Binder ينقله للـ App // الـ Callback (Async) بيحصل بالعكس: // CarService → (Binder) → App's callback thread
⚡ Binder Threading Model

الـ CarService عنده Binder Thread Pool (عادةً 16 thread). كل incoming call من app بياخد thread. لكن الـ VHAL calls من CarService بتتم بـ single dedicated thread لضمان الترتيب. عمر ما تعمل blocking operation في الـ Binder thread — بتعطل threads تانية.

🔢 VehiclePropValue Structure

java — VehiclePropValue — البناء الأساسي // كل property بتتمثل بـ VehiclePropValue class VehiclePropValue { int prop; // Property ID (e.g., 0x11600207 = PERF_VEHICLE_SPEED) int areaId; // Zone (0=global, specific seat/door/window) long timestamp; // Nanoseconds since boot int status; // AVAILABLE / UNAVAILABLE / ERROR RawPropValues value; // الـ actual value } class RawPropValues { int[] int32Values; // ENUM, boolean, int props float[] floatValues; // Speed, temp, percentage long[] int64Values; // Timestamps, odometer byte[] bytes; // Raw binary data String stringValue; // VIN, model name }

🏷️ Property ID Encoding

— فهم الـ Property ID (32-bit integer) // Property ID = [Type (8 bits)] [Area (8 bits)] [Unique ID (16 bits)] // // مثال: PERF_VEHICLE_SPEED = 0x11600207 // 0x11 = VehiclePropertyType.FLOAT (النوع) // 0x60 = VehicleArea.GLOBAL (المنطقة) // 0x0207 = الـ unique ID // // أنواع البيانات: // 0x01 = STRING 0x04 = BOOLEAN 0x10 = INT32 // 0x11 = FLOAT 0x12 = INT64 0x21 = INT32_VEC // 0x22 = FLOAT_VEC 0x41 = MIXED // // أنواع المناطق (Areas): // 0x01 = GLOBAL (كل السيارة) // 0x04 = WINDOW // 0x05 = MIRROR // 0x06 = SEAT // 0x08 = DOOR // 0x10 = WHEEL // فحص النوع من الـ ID: int getAreaType(int prop) { return prop & 0x0f000000; } int getValueType(int prop) { return prop & 0x00ff0000; }

⚡ Reactive: ViewModel + LiveData

MVVM مع Vehicle Properties

class SpeedViewModel extends AndroidViewModel { private final MutableLiveData<Float> mSpeed = new MutableLiveData<>(0f); private Car mCar; private CarPropertyManager mPropMgr; private final CarPropertyManager.CarPropertyEventCallback mSpeedCallback = new CarPropertyManager.CarPropertyEventCallback() { @Override public void onChangeEvent(CarPropertyValue val) { if (val.getStatus() == CarPropertyValue.STATUS_AVAILABLE) { // Post to main thread mSpeed.postValue((Float) val.getValue()); } } @Override public void onErrorEvent(int p, int z) { mSpeed.postValue(-1f); } }; public LiveData<Float> getSpeed() { return mSpeed; } public void connectToCarService() { mCar = Car.createCar(getApplication(), (c, e) -> { if (e == Car.CAR_WAIT_FOR_SERVICE_READY) { mPropMgr = (CarPropertyManager) c.getCarManager(Car.PROPERTY_SERVICE); mPropMgr.registerCallback(mSpeedCallback, VehiclePropertyIds.PERF_VEHICLE_SPEED, 2f); // 2Hz } }); mCar.connect(); } @Override protected void onCleared() { if (mPropMgr != null) mPropMgr.unregisterCallback(mSpeedCallback); if (mCar != null) mCar.disconnect(); } }

Binder Thread Pool

VHAL Caching Strategy

Sample Rate Optimization
